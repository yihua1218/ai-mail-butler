# 角色存取控制 (RBAC)

AI Mail Butler 實作了嚴格的角色存取控制系統 (Role-Based Access Control)，以確保資料隱私與安全的系統管理。系統內的使用者被劃分為三種不同的角色：

## 1. 管理員 (Admin)

系統管理員擁有 AI Mail Butler 平台的最高存取權限。

- **身分認定**：透過比對登入信箱與 `.env` 設定檔中的 `ADMIN_EMAIL` 環境變數來確認身分。
- **存取權限**：
  - **儀表板 (Dashboard)**：能看見全域的系統統計數據（總註冊人數、總接收信件數、總回覆數等）。
  - **信件紀錄**：能瀏覽系統內**所有**被處理過的信件紀錄，無論該信件屬於哪一位使用者。

*若要設定管理員，請在 `.env` 檔案中加入以下設定：*
```env
ADMIN_EMAIL=admin@example.com
```

## 2. 一般使用者 (Registered User)

將自己的信件轉寄給 AI Mail Butler 處理的正常使用者。

- **身分認定**：透過 Magic Link 成功登入並存在於資料庫中，但其信箱不等於 `ADMIN_EMAIL` 的用戶。
- **存取權限**：
  - **儀表板 (Dashboard)**：只能看見個人的專屬儀表板。
  - **信件紀錄**：**僅能看見屬於自己的信件紀錄**。一般使用者絕對無法看見全站的統計數據或任何其他人的信件。

## 3. 匿名使用者 (Anonymous)

尚未登入的訪客。

- **身分認定**：未帶有有效 Magic Link 登入憑證的網頁訪客。
- **存取權限**：
  - **儀表板 (Dashboard)**：只能看見乾淨的歡迎頁面與登入提示。
  - **信件與數據**：完全禁止存取。系統不會向未經驗證的訪客暴露任何信件內容或系統統計數據。
