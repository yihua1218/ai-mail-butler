# 法規補強計劃（台灣 + 美國 + 紐約市）

本文件整理目前系統為提升法規遵循準備度，建議補齊的產品與工程控制。

## 範圍說明
- 本文件屬工程落地清單，不構成法律意見。
- 最終法規適用與解釋仍需由合格法律顧問確認。

## 台灣（以個資法重點為主）

### 目前缺口
1. 同意紀錄缺少可追溯之條款版本化稽核證據。
2. 資料主體權利流程尚不完整（目前主要有刪除，其他權利不足）。
3. 尚未對所有個資資料表建立政策化保存期限。

### 建議補強
1. 新增 `consent_audit_log`（條款版本、來源、時間戳、操作者資訊）。
2. 補齊 DSAR API：存取/匯出/更正/限制處理/撤回同意。
3. 新增保存期限排程清理機制與 legal hold 例外。
4. 建立資料清冊與目的限制對照（按資料表/欄位）。

## 美國（州法常見隱私要求）

### 目前缺口
1. 尚未提供明確的「Do Not Sell/Share」控制與揭露接口。
2. 跨境傳輸目的地與處理者揭露未系統化輸出。
3. 未成年人資料保護機制尚未成為一級控制。

### 建議補強
1. 新增「Do Not Sell/Share」偏好欄位與 API。
2. 新增對外隱私揭露 API（處理者、資料流向、目的地）。
3. 新增年齡閘門與監護人同意擴充點。
4. 建立資料處理目的登錄表（供稽核追蹤）。

## 紐約市 / 紐約州

### 目前缺口
1. 仍需更明確落地 NY SHIELD 對應的基線控制。
2. 尚未建立 NYC AEDT 敏感用途（如就業決策）禁用或控管機制。

### 建議補強
1. 強化靜態資料加密與角色存取稽核。
2. 建立事件應變手冊與演練排程。
3. 在未建立 AEDT 合規流程前，明確禁止就業決策用途。
4. 訓練資料匯出需記錄核准人與使用目的。

## 優先路線圖

### Phase A（立即）
1. 同意稽核表與 DSAR 匯出 API。
2. `transcripts/feedback/logs` 保存期限清理排程。
3. 訓練匯出存取與核准紀錄。

### Phase B（下一步）
1. Do Not Sell/Share 控制與使用者揭露端點。
2. 跨境傳輸揭露與處理者清單輸出。
3. 事件應變手冊與演練紀錄。

### Phase C（後續）
1. 未成年人資料處理與監護人同意流程。
2. 若導入就業相關用途，再啟動 NYC AEDT 準備模式。

## 驗證清單
1. 同意閘道與資料保留刪除路徑單元測試。
2. DSAR 權限邊界與流程 API 測試。
3. 匯出脫敏與存取稽核安全回歸測試。
4. 每月或每季輸出遵循證據報表。
